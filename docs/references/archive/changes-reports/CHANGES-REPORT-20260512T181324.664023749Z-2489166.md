## New Additions To Consider

While this refresh introduced no new upstream content, reading the refreshed reference corpus reveals capabilities and patterns the project has not yet implemented. These are prioritized by how directly they map to existing crates:

### 1. ACP Rust SDK v1 — SACP component model (`agentcast-acp`, `agentcast-runtime`)

The `rfds-rust-sdk-v1.md` reference describes a proposed replacement for the `agent-client-protocol` crate, based on SACP (Symposium ACP). Key design patterns that should inform `agentcast-acp` and `agentcast-runtime`:

- Component-based builder API instead of fixed trait callbacks
- Context parameter passed to handlers, enabling in-handler message sends (critical for streaming `session/update` notifications during prompt processing)
- Dynamic per-session handler registration using closures
- Typed extension method/notification support (`_` prefix methods)
- First-class proxy support compatible with the conductor pattern in `agentcast-gateway`

Reference: `docs/references/acp/docs/markdown/0055-agentclientprotocol-com-rfds-rust-sdk-v1.md`

### 2. ACP Proxy / Conductor pattern (`agentcast-gateway`, `agentcast-runtime`)

The `rfds-proxy-chains.md` RFD describes the conductor as the central orchestrator for proxy chains. `agentcast-gateway` should implement the conductor role: routing `proxy/successor` messages between proxy components, agent, and MCP servers.

Reference: `docs/references/acp/docs/markdown/0050-agentclientprotocol-com-rfds-proxy-chains.md`

### 3. MCP-over-ACP transport (`agentcast-mcp`)

`rfds-mcp-over-acp.md` proposes `"transport": "acp"` as a new MCP transport type. This enables client-side MCP servers that communicate over the existing ACP channel, eliminating the need for side-channel TCP/HTTP ports. `agentcast-mcp` should implement this ACP-transport bridge.

Reference: `docs/references/acp/docs/markdown/0051-agentclientprotocol-com-rfds-mcp-over-acp.md`

### 4. ACP Agent Registry manifest format (`agentcast-registry`)

The `rfds-acp-agent-registry.md` defines the `<id>/agent.json` manifest schema: required fields (`id`, `name`, `version`, `description`, `distribution`), distribution types (`binary`, `npx`, `uvx`), icon requirements (SVG, 16×16, `currentColor`), and the `agentclientprotocol/registry` repo as the canonical manifest store. `agentcast-registry` should implement ingestion and serving of these manifests.

Reference: `docs/references/acp/docs/markdown/0023-agentclientprotocol-com-rfds-acp-agent-registry.md`

### 5. Session Config Options generalization (`agentcast-config`, `agentcast-server`)

The `rfds-session-config-options.md` and `protocol/session-config-options.md` describe replacing hard-coded mode/model selectors with a flexible key-value config selector mechanism. `agentcast-config` should model `SessionConfigOption` types, and `agentcast-server` should surface them in `session/new` responses. Note: the reference explicitly deprecates `session/set_mode` in favor of this mechanism.

References: `docs/references/acp/docs/markdown/0024-agentclientprotocol-com-rfds-session-config-options.md`, `docs/references/acp/docs/markdown/0061-agentclientprotocol-com-protocol-session-config-options.md`

### 6. Session List endpoint (`agentcast-server`, `agentcast-core`)

`protocol/session-list.md` describes a stable `session/list` method returning `sessionId`, `cwd`, `title`, `updatedAt`, and `_meta`. This is announced as stabilized (`announcements/session-list-stabilized`). `agentcast-server` should implement this endpoint and `agentcast-core` should define the `SessionMetadata` type.

Reference: `docs/references/acp/docs/markdown/0063-agentclientprotocol-com-protocol-session-list.md`

### 7. ACP v2 protocol tracking (`agentcast-protocol`)

`rfds-v2-overview.md` tracks the ACP v2 breaking-change roadmap. Current accepted drafts targeting v2: new prompt lifecycle, message IDs, session fork from specified IDs, remote transports (streamable HTTP + WebSocket). `agentcast-protocol` should track these as pending v2 work. The `agent-client-protocol = { version = "0", features = ["unstable"] }` Cargo dependency already signals willingness to adopt unstable features.

Reference: `docs/references/acp/docs/markdown/0027-agentclientprotocol-com-rfds-v2-overview.md`

### 8. MCP specification 2025-11-25 and draft features (`agentcast-mcp`)

The `mcp/docs/markdown/` corpus now includes the full `2025-11-25` specification and draft specs. Notable draft additions relevant to `agentcast-mcp`: tasks (background work tracking), subscriptions (resource change notifications), MRTR (multi-resource transactional reads), and sessionless MCP. The `rmcp = { version = "1.6" }` dependency should be evaluated against these.

Reference: `docs/references/mcp/docs/markdown/` (specification-2025-11-25-* and specification-draft-* files)

### 9. Elicitation protocol (`agentcast-server`, `agentcast-acp`)

The `rfds-elicitation.md` (32 KB) describes a mechanism for agents to request structured user input mid-session. Both ACP elicitation and MCP elicitation (already in `rmcp` features list) should be supported. The `agentcast-acp` crate with the `elicitation` rmcp feature already signals intent.

Reference: `docs/references/acp/docs/markdown/0039-agentclientprotocol-com-rfds-elicitation.md`

### 10. Claude Agent SDK patterns for `agentcast-cli` (`agentcast-cli`)

The `claude-code/markdown/` corpus covers Agent SDK sessions, subagents, hooks, streaming output, and file checkpointing patterns. `agentcast-cli` should use these patterns when building the end-user CLI interface to the agentcast stack.

References: `docs/references/claude-code/markdown/` (agent-sdk-* files)
